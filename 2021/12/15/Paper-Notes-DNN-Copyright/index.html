Copy, Right? A Testing Framework for Copyright Protection of Deep Learning Models

DNN Copyright Issues

DNN版权的三种常见威胁：

  1. 模型微调 (Model Fine-tuning)
  2. 模型剪枝 (Model Pruning)
  3. 模型提取 (Model Extraction)

现有的保护技术（大多基于水印技术）的缺点：

  • 具侵入性 (invasive)，往往需要在模型训练部分做手脚，这可能会影响模型的实用性或给模型带来新的安全风险；
  • 容易遭受自适应 (adaptive) 攻击 (试图移除/替换水印或以敌对方式阻止水印检索的自适应攻击。)
  • 对模型提取(Model Extraction)攻击的防御效果不佳。

$DEEPJUDGE$ 测试框架

文章提出的 $DEEPJUDGE$ 测试框架的优点：

  • 不具侵入性，不干扰训练过程。

  • 高效，仅需要过少量测试用例并快速扫描受害模型&可疑模型。

  • 灵活，它可以很容易地结合新的测试指标或测试用例生成方法，以获得更自信/稳健的判断。

  • 应对自适应攻击和模型提取攻击的能力有所增加。

作者使用了图像分类和语音识别数据集在多种模型结构上进行了大量实验，并验证了 $DEEPJUDGE$ 在三种典型版权侵权场景下的有效性（模型微调、剪枝和提取）。

$DEEPJUDGE$ 测试框架结构：

  1. 测试用例生成 (test case generation)
  2. 多级测试指标 (a set of multi-level distance metrics for testing)
  3. Final judgement step: a thresholding and voting based judgement mechanism.

这段伪代码描述了 $DEEPJUDGE$ 的完整过程。它将：受害模型 $O$、可疑模型 $S$、以及与受害模型关联的一组数据集 $D$ 作为输入，返回被用作证据的测试指标(testing metrics)值以及判断结果。

数据集 $D$ 可以由所有者从受害者模型的训练集或测试集提供。

  1. 前两步为测试用例生成步骤，这里先从输入数据集 $D$ 中选择一组seed。

  2. 利用这组seed生成一组极端测试用例(extreme test cases)。

  3. 根据生成的测试用例，$DEEPJUDGE$ 计算可疑模型和受害者模型之间由测试指标得到的差异性分数。

  4. 通过阈值和投票机制，根据受害者模型与一组negative可疑模型之间的差异性得分，最终判断可疑模型是否侵害了受害者版权。

测试指标 testing metrics

默认可以访问受害模型 $O$ 的全部细节，包括训练集、测试集、甚至训练细节。
该框架通过多级 testing metrics 对 DNN 进行 evidence-based 的所有权验证，这些 testing metrics 相互补充以得出更可靠的判断。

  • Property-level: 通过对抗攻击（例如 FGSM 和 PGD ）来对模型 input 进行轻微扰动，若经扰动得出的 output 和原本的 label 不同，则证明攻击成功。由此，能够计算出模型的 Robustness Distance (RobD)
    • 可被视作模型的一种 fingerprint。
    • 需要最少的 Knowledge - 仅需要 output label。
  • Neuron-level: 通过神经元的输出状态定义了两个指标：Neuron Output Distance (NOD) & Neuron Activation Distance (NAD)，NOD 计算两个模型中神经元的平均输出差异值；NAD计算被激活神经元的个数差异，如果激活则返回1。
    • 适用于白盒测试。
    • 层索引 $l$
  • Layer-level：通过层输出定义了三个指标：Layer Output Distance (LOD), Layer Activation Distance (LAD) & Jensen-Shanon Distance (JSD)，LOD 测量两个模型层输出之间的 $L_p$ 范数距离；LAD 测量模型同一层内被激活神经元的平均个数差异（平均 NAD）；JSD 是一种度量概率相似性的指标，它量化两个模型输出分布的相似性。
    • JSD 在模型提取攻击的场景下尤为强大（可疑模型为基于受害模型返回的概率向量（分布）提取）
    • $L_p$ 范数，实验中 $p=2$

测试用例生成

黑盒设置：仅能访问可疑模型的输入和输出。

  • 使用现有的对抗学习方法（FGSM, PGD, CW）对受害模型的 input 的输入进行扰动，生成被扰动的数据集，作为测试集。
  • 经 Model Fine-tuning 和 Pruning 的到的可疑模型都会有和受害模型相似的决策边界；
  • 经 Model Extraction 的模型也依赖于受害模型返回的概率向量，因此也会和受害模型的决策边界越来越相似。因此：模型提取的越好，越容易被基于决策便捷的测试识别。
